## 2) 레이어별 상세 설계 포인트

### A. 경계/진입(Edge): API Gateway / Ingress

* **인증/인가**: AAD(OAuth2/OIDC), Managed Identity(서버→서버)
* **레이트 리밋/쿼터**: 남용 방지, 비용 제어
* **감사/로깅 헤더**: 추적 ID(correlation-id), 사용자/역할

### B. 코어 런타임: Agent Runtime (ChatAgent)

* **역할/지침(Instructions & Policies)**: 업무 규칙, 금칙어, 개인정보 취급
* **플래너/라우터(Planner/Router)**: 자연어 의도 → Tool 호출 계획 수립 → 실행 순서 제어
* **메모리/스레드(Memory/Thread)**: 세션 상태, 사용자 선호, 컨텍스트 지속성
* **비동기(Async) 실행**: 고지연 I/O(검색·API)를 자연스럽게 기다리며 동시성 확보

> 핵심은 “**자연어 → 행동(액션)으로의 안전한 변환**”.
> LLM이 **필요한 툴만** 호출하고, 결과를 검증·요약해 응답으로 반환.

### C. 툴 계층: Tool Orchestration

* **Function Calling 모델**을 통해 LLM이 함수를 “스스로” 호출
* **샌드박스/Scope**: Agent별 툴 접근 범위, 입력/출력 검증, 속도/시간 제한
* **예시 툴**:

  * 사내 검색(`search_docs(query)` → Azure AI Search)
  * 데이터 조회(`get_customer(id)` → SQL/Cosmos)
  * 문서 생성/메일 전송(템플릿 기반 자동화)
  * 외부 API 호출(그래프/SharePoint/ServiceNow 등)

### D. LLM / Reasoning

* **Azure OpenAI** 또는 승인이 끝난 **OSS 모델**(온프렘·전용 VNet) 사용
* **프롬프트 체인**: System/Developer/User → Guard → Plan → Tool → Synthesize
* **토큰/비용 관리**: 요청 크기 제한, 압축 요약(chunking), 캐시

### E. 세이프티/거버넌스: Safety & Guardrails

* **프롬프트 가드**: 프롬프트 인젝션/탈출 방지
* **정책 필터**: 업종/기관별 금지 주제, PII/민감정보 차단
* **출력 검증**: 형식/스키마, 수치 범위, 역할 일관성

### F. 데이터/지식 평면

* **RAG(검색 증강)**: Azure AI Search + Vector DB(임베딩)

  * 소스: 정책/지침, 매뉴얼, SOP, 티켓 이력, 보고서
  * 파이프라인: 수집→전처리→분할→메타데이터→인덱스→증강 프롬프트
* **운영 데이터**: SQL/Cosmos/Blob/Data Lake(보안 규정 준수)
* **외부 API/서비스**: Microsoft Graph/SharePoint/Confluence/ServiceNow/Functions

### G. 플랫폼/운영

* **Observability**:

  * AppInsights(로그/트레이스/지연/오류), Tool 호출/토큰 사용량 대시보드
  * 프롬프트/검색 쿼리 샘플링 → 품질 분석(헬륨/가드레일 튜닝)
* **보안/ID**:

  * AAD, Managed Identity, 키 보관(Key Vault), 네트워크 경계(VNet/Private Link)
* **배포/런타임**:

  * Azure Functions(서버리스), Container Apps(마이크로서비스), AKS(대규모/하이브리드), On-prem(데이터 주권)

## 3) 동작 시나리오(엔드-투-엔드)

1. **사용자**가 “지난 분기 영업 리포트 요약해서 메일로 보내”라고 입력
2. **Gateway**가 인증, 레이트 리밋, 감사 ID 부여
3. **Agent Runtime**이 의도 파악 → 계획 수립(검색→분석→생성→메일)
4. **RAG**로 관련 문서 검색(메타데이터 필터: 기간·부서·보안등급)
5. **LLM**이 요약/표 생성, 포맷 검증(스키마)
6. **Tool**로 메일 전송(승인 정책·수신자 화이트리스트 체크)
7. **Safety**가 결과의 민감정보/외부 전송 정책 점검
8. **Observability**에 전 과정 로그(프롬프트/툴/시간/비용) 기록
9. **사용자**에게 결과/상태(성공·오류·부분성공) 응답

## 4) 개발자 관점 핵심 API/패턴

### 최소 예제(Async)

```python
from agent_framework import ChatAgent, tool
from agent_framework.azure import AzureAIAgentClient
from azure.identity.aio import DefaultAzureCredential

# 1) Tool 정의
@tool
async def search_docs(query: str) -> list[dict]:
    # Azure AI Search 호출(의사 코드)
    # results = ais_client.search(query, top=5)
    # return [{"title": r.title, "url": r.url, "snippet": r.snippet} for r in results]
    return [{"title": "Sample", "url": "https://contoso", "snippet": "demo"}]

# 2) Agent 실행
async def main():
    async with (
        DefaultAzureCredential() as cred,
        ChatAgent(
            chat_client=AzureAIAgentClient(async_credential=cred),
            instructions=(
                "You are an enterprise assistant. "
                "Use 'search_docs' for factual answers and include sources."
            ),
            tools=[search_docs],  # 명시적 도구 주입
        ) as agent
    ):
        res = await agent.run("분기 매출 보고서 요약해줘. 출처도 알려줘.")
        print(res.output_text)

# if __name__ == "__main__": asyncio.run(main())
```

**포인트**

* Tool은 **명확한 시그니처/입출력 스키마**를 갖고, 내부에서 인증·예외 처리
* Agent `instructions`에 **도구 사용 규칙**과 **안전/정책 요구사항**을 명확히 명시
* 운영 시에는 **프롬프트/툴 호출 로그**를 샘플링·분석하여 지속 개선

## 5) 보안·거버넌스 체크리스트

* **프롬프트 가드/정책**: 민감정보/규정 대응 문구, 금칙 표현, 출처 강제
* **도구 권한 분리**: 읽기/쓰기, 사내망/외부망, 상위권한 금지
* **데이터 경계**: 테넌트/부서/프로젝트 경계·마스킹·필드 수준 보안
* **감사 가능성**: 누가·언제·무엇을·어디로(로그 필수)
* **비용 경계**: 토큰/쿼리 크기 제한, 캐시/요약 재사용
* **회복력**: 타임아웃/재시도/폴백(대체 검색/간단 요약)

---

## 6) 운영(Observability) 베스트 프랙티스

* **세션/툴/프롬프트** 단위 코릴레이션-ID로 전체 추적
* **품질 지표**: 해결율, 재질문율, RAG 히트율, 인용 정확도, 평균 지연
* **비용 지표**: 요청당 토큰/Tool 호출 횟수/캐시 적중률
* **릴리즈 검증**: 프롬프트/정책/인덱스 변경 시 A/B 또는 샌드박스 트래픽

## 7) 배포 옵션 가이드

| ------------------- | -------------------- | ------------ |
